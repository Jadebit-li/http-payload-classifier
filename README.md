# AI-WAF: Malicious Payload Detector

A binary ML classifier that detects malicious HTTP payloads using character-level TF-IDF and Logistic Regression. Built as a simplified version of the detection layer inside a Web Application Firewall.

## What it does

Takes a raw HTTP payload string — something typed into a search box, login form, or URL parameter — and predicts whether it's malicious or benign, with a confidence score.
Detects: SQL Injection, Cross-Site Scripting (XSS), Path Traversal, Command Injection.

## Why character-level TF-IDF

Standard word-level tokenizers fail on attack payloads because the malicious signal lives in character patterns — ' OR, <script>, ../ — not dictionary words. Character n-gram vectorization (ngram_range=(1,4)) captures these structural patterns directly as numerical features.

## Pipeline

* Raw CSV → dedup check → stratified train/test split (80/20)
* TF-IDF fit on training data only
* Logistic Regression training
* Evaluation on held-out test set
* Save model + vectorizer as .pkl
* Streamlit UI loads .pkl and classifies new input

---

## Data leakage prevention

* Duplicate check before splitting — confirmed zero duplicate payloads across 31,067 rows
* Stratified split — preserves attack-type proportions across train/test, critical for minority classes (CmdI: only 89 samples)
* TF-IDF fit on training data only — .transform() used on test data, never .fit_transform()

## Dataset Profile

Morzeux HttpParamsDataset — open source, MIT license.

* **Total Clean Rows:** 31,067 (Verified 0 duplicate payloads, 0 missing entries)
* **Benign Class (norm):** 19,304 records (62.13%)
* **Malicious Class (anom):** 11,763 records (37.86%)

### Target Vulnerability Breakdown
* **SQL Injection (SQLi):** 10,852 records
* **Cross-Site Scripting (XSS):** 532 records
* **Path Traversal:** 290 records
* **Command Injection (CmdI):** 89 records

## Limitations
* Dataset is generated by automated tools (sqlmap, xssya) on a single academic benchmark — not real-world traffic. Results reflect pattern memorization of tool-generated payloads, not generalization to novel hand-crafted attacks.
* ML-based classifiers are vulnerable to adversarial evasion — an attacker who knows the model can craft payloads that statistically resemble benign traffic. Real WAFs use rule-based signature matching as the primary layer; ML serves as a supplement.
* This is a classifier, not a WAF. It does not intercept live traffic, parse HTTP headers, or integrate into a request pipeline.
